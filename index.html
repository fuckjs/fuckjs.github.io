防止ajax访问冲突

一个简易的需求，点一个按钮，则向服务器请求资源，不作处理时，多次点击后会有很多个请求在等待。粗暴的解决方式是点一次就将按钮disable掉。还有一种情况，比如加载页面前需要向后台的多个接口调用数据，同时发送两个以上请求；又比如实时搜索，每输入一个字符就要调用一次查询，调用速度很快。这样就会产生问题。

这种情况如果不跨域的话ajax是可以通过设置为同步来解决的。但是在使用jsonp执行跨域请求的时候，如果加载页面前需要向后台的多个接口调用数据，就会比较难解决。做项目的时候自己研究的一个解决方法，通过定时器来查询，仅供参考。思路就是：在前面加一个变量isAjaxing，当ajax开始调用的时候，判断isAjaxing是不是true；是false则说明没有调用，进入交互部分并关闭定时器；如果是true则说明正在调用，return返回。但因为数据必须查询出来加载到页面上，每隔50ms判断一次，直到可以正常交互。

代码如下

var isAjaxing = false;
exports.axjsonp = function(url, data, successfn) {
        data = (data == null || data == "" || typeof(data) == "undefined") ? {
            "date": new Date().getTime()
        } : data;

        var timer=setInterval(queryAjax,50);
        function queryAjax(){
            if(isAjaxing){
                return;
            }
            $.ajax({
                type: "POST",
                url: url,
                data: data,
                //返回数据的格式
                dataType: "jsonp", //"xml", "html", "script", "json", "jsonp", "text".
                jsonp: "callbackparam", //传递给请求处理程序或页面的，用以获得jsonp回调函数名的参数名(默认为:callback)
                jsonpCallback: "success_jsonpCallback", //自定义的jsonp回调函数名称，默认为jQuery自动生成的随机函数名
                //在请求之前调用的函数
                beforeSend: function() {
                    isAjaxing = true;
                    //$("#btn_loading").css('display', '');
                },
                //成功返回之后调用的函数             
                success: function(data) {
                    successfn(data);
                },
                //调用执行后调用的函数
                complete: function(XMLHttpRequest, textStatus) {
                    isAjaxing = false;
                    $("#btn_loading").css('display', 'none');
                },
                //调用出错执行的函数
                error: function(XMLHttpRequest, textStatus, errorThrown) {
                    if (XMLHttpRequest.status != '200') {
                        alert("AJAX请求错误：请求状态码：" + XMLHttpRequest.status + " readyState:" + XMLHttpRequest.readyState + "错误：" + textStatus);
                    }
                }
            });
            clearTimeout(timer);
        }

};

关于jQuery源码的一些思考

一. 最外层的匿名自执行函数为什么传window和undefined?

之所以传window作为参数，有两个原因。

  • 因为window是作用域最顶端，查找速度比较慢
  • 在压缩的时候有用

而之所以传undefined，是因为undefined是可以被人为修改的。这样在内部就可以保证undefiend没有被修改。

二. jQuery中使用异步加载顺序执行?

var dfd  = $.Deferred();
setTimeout(function(){
    alert(1);
    dfd.resolve();
},1000);
dfd.done(function(){
    alert(2);
})

定时器虽然是异步执行，但是此例子中添加了.resolve()方法，是先弹1后弹2的。
